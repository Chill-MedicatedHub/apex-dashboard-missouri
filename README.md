# Chill Medicated · Missouri — Sales Dashboard

A self-contained dashboard for the Missouri Apex Trading "All Product Sales" report.
The scraper pulls data from the **Missouri** Apex account using your browser session;
the dashboard reads the result and renders KPIs, charts, and tables.

This is a twin of the Massachusetts dashboard, pointed at the Missouri account.
The three things that differ from the MA repo:

- Data file is `sales_data_mo.json` (so MA and MO never overwrite each other)
- Labels read "Missouri" instead of "Massachusetts"
- The scraper reads the account ids + brand id from `.env` (MA had them hardcoded)

```
apex-dashboard-mo/
├── scraper.py            ← pulls Missouri data from Apex
├── dashboard.html        ← the dashboard UI
├── sales_data_mo.json    ← generated by scraper (gitignored, don't commit)
├── .env                  ← your Missouri secrets (create from .env.example, gitignored)
├── .env.example          ← template
├── .gitignore
├── requirements.txt
└── logo.png              ← copy this from the MA repo (header references it)
```

## ⚠️ Before your first commit

This repo is set up so secrets and data stay private:

- `.env` is gitignored — it holds a **live session cookie** for your Missouri
  Apex account. Treat it like a password. Never commit it.
- `sales_data_mo.json` is gitignored — it holds **real sales figures**. If this
  repo is public (e.g. for GitHub Pages), an ignored data file keeps your numbers
  off the public internet.

Commit order matters: the `.gitignore` is already in place, so as long as you
don't force-add those files, you're safe. If you ever commit `.env` by accident,
it stays in git history even after deletion — rotate the cookie immediately.

## 1. First-time setup

### A. Install dependencies
```
pip install -r requirements.txt
```

### B. Create your .env
```
# macOS / Linux
cp .env.example .env
# Windows
copy .env.example .env
```

### C. Fill in your MISSOURI account values
Open `.env` and set:

- `APEX_COOKIE` — your Missouri Apex session cookie (see step D)
- `APEX_USER_ID` — the Missouri account's user id
- `APEX_COMPANY_ID` — the Missouri company id
- `APEX_BRAND_ID` — the Missouri brand id for Chill Medicated

You'll find the user/company/brand ids in the same report request payload where
you grab the cookie (the Network tab, in the `run-all-product-sales-report` request).

> Because Missouri is a different login, these ids are almost certainly different
> from the Massachusetts ones. If you leave them blank, the scraper falls back to
> the MA defaults and will pull the wrong account.

### D. Grab your session cookie from Apex
1. Log into the **Missouri** account at <https://app.apextrading.com>
2. Open the **All Product Sales** report
3. Press **F12** → **Network** tab → **Fetch/XHR**
4. Click **Run** on the report
5. Click the **`run-all-product-sales-report`** request
6. **Headers** → **Request Headers** → copy the entire **`Cookie:`** value
7. Paste it between the quotes in `.env`: `APEX_COOKIE="..."`

---

## 2. Run it
```
# 1. Pull data
python scraper.py

# 2. Start a local server
python -m http.server 8000

# 3. Open http://localhost:8000/dashboard.html
```

---

## 3. When the cookie expires
You'll see a 401/403 error in the terminal. Re-grab a fresh cookie (step 1.D)
and paste it into `.env`. Takes about 30 seconds.

---

## Troubleshooting

**"No data yet" in the dashboard** — the scraper hasn't produced `sales_data_mo.json`
yet, or you opened the file directly via `file://`. Run `python scraper.py`, then
serve via `python -m http.server 8000`.

**Scraper returns 0 rows** — usually a wrong `APEX_BRAND_ID` or a date range with no
sales. Double-check the Missouri brand id, and try widening `APEX_DAYS_BACK`.

**Scraper returns 401/403** — cookie expired or from the wrong account. Re-grab it.

**Logo missing in header** — copy `logo.png` from the MA repo into this folder.
